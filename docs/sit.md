# /sit - 提交到 SIT 环境

自动将当前分支的改动提交并合并到 SIT 环境。

## 功能

- 自动检测最新的 `F_*` 远程分支
- 根据代码改动自动生成规范的 commit message
- 从 F 分支拉取最新代码
- 推送当前分支并合并到 sit 分支

## 使用方法

```
/sit
```

## 执行流程

```
┌─────────────────────────────────────┐
│  1. 检查当前状态                      │
│     - git status                     │
│     - 检测最新 F_* 分支               │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  2. 自动提交改动（如有）              │
│     - 分析 diff 生成 commit message  │
│     - git add . && git commit        │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  3. 同步 F 分支                      │
│     - git fetch origin               │
│     - git pull origin F_xxx          │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  4. Push 当前分支                    │
│     - git push origin <当前分支>     │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│  5. Merge 到 sit                     │
│     - git checkout sit               │
│     - git merge <原分支>             │
│     - git push origin sit            │
│     - git checkout <原分支>          │
└─────────────────────────────────────┘
```

## Commit Message 规范

自动生成的 commit message 遵循 Conventional Commits 规范：

```
<type>(<scope>): <description>
```

| Type | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| refactor | 代码重构 |
| docs | 文档更新 |
| style | 代码格式调整 |
| test | 测试相关 |
| chore | 构建/工具相关 |

## 注意事项

- 如遇冲突，会提示手动解决后再继续
- 执行完成后会自动切回原分支
- 建议在 feature 分支上使用
